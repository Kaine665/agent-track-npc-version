## 2. 开发策略

### 2.1 核心原则

**前端独立开发，不依赖后端 API**

- 使用 Mock API 模拟后端响应
- 通过 API 适配层统一接口
- 支持 Mock/真实 API 无缝切换
- 减少受后端 API 变更影响

### 2.2 开发流程

```
原型设计 → UI库选择 → API适配层框架 → 路由配置 → 基础组件 → 页面开发（按需实现Mock） → 对接真实API
```

### 2.3 Mock 和适配层的实现策略

**推荐方案：框架提前搭建，Mock 按需实现**

**为什么这样设计**：

1. **API 适配层框架提前搭建**：

   - 定义统一的接口规范
   - 实现模式切换机制
   - 页面开发时直接使用统一接口
   - **优势**：避免重复工作，统一规范

2. **Mock 数据按需实现**：
   - 开发某个页面时，需要什么 API 就实现什么 Mock
   - 边开发边调整 Mock 数据格式
   - **优势**：只实现需要的，避免过度设计，更符合实际需求

**实现时机**：

- **阶段 8**：搭建 API 适配层框架（接口定义、模式切换、基础结构）
- **阶段 10**：开发 NPC 列表页时，实现 `api.agents.getList()` 的 Mock
- **阶段 11**：开发创建 NPC 页时，实现 `api.agents.create()` 的 Mock
- **阶段 12**：开发对话页时，实现 `api.messages.send()` 和 `api.history.get()` 的 Mock

**优势**：

- ✅ 符合人类开发习惯（需要什么做什么）
- ✅ 避免实现暂时用不到的功能
- ✅ 开发页面时更清楚需要什么数据格式
- ✅ 可以边开发边调整 Mock 数据
- ✅ 框架提前搭建，避免重复工作

#### 任务 8.2：实现 API 适配层

**任务描述**：

- 创建 API 适配层（适配器模式）
- 实现统一的 API 调用接口
- 支持 Mock 模式和真实 API 模式切换

**文件结构**：

```
src/
├── api/
│   ├── index.js          # API入口，模式切换
│   ├── adapter.js        # 适配器接口定义
│   ├── mockAdapter.js    # Mock适配器实现
│   └── httpAdapter.js    # HTTP适配器实现（真实API）
```

**设计模式**：适配器模式（Adapter Pattern）

**API 接口设计**（统一接口，不依赖具体实现）：

```javascript
// 业务代码使用统一的API接口
api.agents.create(data);
api.agents.getList(userId);
api.agents.getById(agentId);
api.messages.send(data);
api.history.get(userId, agentId);
api.sessions.getList(userId);
```

**模式切换**（通过环境变量）：

```javascript
// 开发环境：使用Mock
const api = createApi("mock");

// 生产环境：使用真实API
const api = createApi("http");
```

**优势**：

- 统一接口，业务代码不关心是 Mock 还是真实 API
- 方便切换，通过环境变量控制
- 减少受后端 API 限制，适配层可以适配后端 API 差异

**验收标准**：

- [ ] `src/api/` 目录已创建
- [ ] API 适配器接口已定义
- [ ] Mock 适配器已实现（调用 Mock API）
- [ ] HTTP 适配器已实现（占位，后续对接真实 API）
- [ ] 可以通过环境变量切换 Mock/真实 API
- [ ] 所有业务代码使用统一的 API 接口

**预估时间**：3-4 小时

## 9. 关键设计说明

### 9.1 Mock API 设计

**目的**：前端独立开发，不依赖后端

**实现方式**：

- 创建 Mock 服务，模拟所有后端 API
- Mock 数据符合 API 设计文档格式
- 支持模拟延迟和错误情况

### 9.2 API 适配层设计

**目的**：统一接口，减少受后端 API 限制

**设计模式**：适配器模式（Adapter Pattern）

**实现方式**：

- 定义统一的 API 接口
- Mock 适配器：调用 Mock API
- HTTP 适配器：调用真实后端 API
- 通过环境变量切换适配器

**优势**：

- 业务代码不关心是 Mock 还是真实 API
- 后端 API 变更时，只需修改适配器
- 方便测试和开发
